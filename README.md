DebugerW — это портативная утилита для автоматического обнаружения скрытых майнеров, стилеров паролей и бэкдоров (удалённого управления) на компьютерах под управлением Windows. Программа работает в реальном времени, отслеживает запуски/завершения процессов, анализирует ошибки в журналах и сигнализирует о подозрительной активности.

🧠 Возможности
Детекция майнеров — по имени процесса, характерным аргументам командной строки (stratum+tcp://, --algo, --url, --user, --pass), высокому потреблению CPU и запуску из временных папок.

Детекция стилеров паролей — по имени (redline, azorult, raccoon и др.) и ключам (--password-store, --cookies, --logins, --dump).

Детекция бэкдоров и RAT — по имени (ngrok, teamviewer, anydesk, tightvnc и др.) и аргументам (-reverse, -bind, -listen, --shell), а также по сетевым соединениям на подозрительные порты (4444, 5555, 6666 и другие).

Мониторинг процессов — показывает запуск и завершение каждого процесса с полной командной строкой.

Отслеживание системных ошибок — читает журналы Application и System, выводит ошибки с EventID и описанием.

Минимизация ложных срабатываний — встроен белый список системных и легитимных процессов, а также безопасных путей (Windows, Program Files, Defender).

Работает без установки Python — собирается в единственный EXE-файл, который можно запустить на любом ПК с Windows.

Автозапуск — программа может быть добавлена в Планировщик задач для запуска при входе в систему с правами администратора.

Логирование — все события записываются в текстовый файл рядом с программой.

⚙️ Требования
Операционная система: Windows 7 / 8 / 10 / 11 (32‑бит или 64‑бит).

Права администратора (для чтения системного журнала и списка процессов).

🚀 Использование
Запустите DebugerW.exe от имени администратора (двойной клик → подтвердите UAC).

При первом запуске программа предложит добавить себя в автозагрузку через Планировщик задач (рекомендуется).

Программа начнёт мониторинг в реальном времени. В консоли будут появляться строки с событиями.

При обнаружении подозрительного процесса вы увидите яркое предупреждение с типом угрозы ([МАЙНЕР], [СТИЛЕР], [БЭКДОР]) и подробными причинами.

Для остановки нажмите Ctrl+C в окне консоли.

📄 Формат вывода
Пример сообщения о подозрительном процессе:
text
[2026-07-08 14:32:15.123] [ВНИМАНИЕ] [МАЙНЕР]
  PID: 12345
  Имя: xmrig.exe
  Путь: C:\Users\Public\temp\xmrig.exe
  Команда: xmrig.exe -o stratum+tcp://pool.com:4444 -u user -p x
  Причины: stratum-параметры: stratum+tcp://, -o
Также в лог-файл (debug_log_YYYYMMDD_HHMMSS.txt) записываются все события, включая запуски/завершения процессов и ошибки журналов.

🙏 Благодарности
Библиотеки: pywin32, wmi, psutil, PyInstaller.

Вдохновение — реальные случаи скрытого майнинга на серверах.
